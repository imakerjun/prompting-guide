# 발표자 노트: 에이전트 시스템

#### 슬라이드 1: 도입부 — 자율적으로 일하는 AI

[오프닝 — 브릿지와 개념 도입 (2분)]

지난 두 시간을 돌아보면요.
9강에서 AI가 **행동**하게 만드는 법을 배웠고,
10강에서 AI가 **생각**하게 만드는 법을 배웠어요.

그러면 사고와 행동을 합치면 뭐가 될까요?

**생각하고 → 행동하고 → 결과를 보고 → 다시 생각하고 → 또 행동하고...**
이 루프를 스스로 반복하면서 자율적으로 일하는 AI.
이게 바로 에이전트 시스템이에요.

📌 화면: "에이전트 시스템" 페이지 제목

직장인분들에게 가이드를 드리면요.
이 챕터는 자율 AI 에이전트를 구축하는 개발자를 위한 고급 내용이에요.
Claude.ai를 쓰시는 분들은 **"연구와 정보 수집"** 섹션의
구조화된 검색 프롬프트를 참고하세요.
복잡한 조사 업무에 바로 활용할 수 있습니다.

---

#### 슬라이드 2: 장기적 추론과 상태 추적

[에이전트의 핵심 능력 (2분)]

📌 화면: 장기적 추론과 상태 추적 섹션

에이전트에게 가장 중요한 능력이 뭘까요?
**방향감을 잃지 않는 것**이에요.

복잡한 작업은 시간이 오래 걸리잖아요.
중간에 "내가 뭘 하고 있었지?" 하고 길을 잃으면 안 되거든요.

Claude의 최신 모델은 탁월한 상태 추적 능력을 가지고 있어요.
한 번에 모든 것을 해결하려 하지 않고,
**점진적으로 진전**하면서 전체 방향을 유지합니다.

그리고 맥락 인식 기능이 있어서
대화가 길어져도 남은 컨텍스트 윈도우를 추적할 수 있어요.
"토큰이 부족하니까 여기서 멈춰야지" 같은 판단을 스스로 하는 거예요.

#### 슬라이드 3: 자율성과 안전성의 균형

[핵심 — 안전 가드레일 (3분)]

📌 화면: 자율성과 안전성 경고 Callout

자, 여기서 중요한 이야기를 해야 해요.

에이전트가 자율적으로 일한다는 건 좋은데,
가이드 없이 놔두면 무슨 일이 벌어질까요?

Claude Opus 4.6은 지시 없이도
파일 삭제, 강제 푸시, 외부 서비스에 게시 같은
**되돌리기 어려운 작업**을 수행할 수 있어요.

그래서 안전 가드레일이 필수입니다.

📌 화면: 안전 가드레일 프롬프트 코드 블록

이 프롬프트를 보세요.
핵심은 **가역성**이에요.

"행동의 가역성과 잠재적 영향을 고려하세요."

파일 편집이나 테스트 실행 같은 **되돌릴 수 있는 작업**은 자유롭게.
하지만 파일 삭제, 강제 푸시, PR 코멘트 같은 **되돌리기 어려운 작업**은
반드시 사용자에게 확인을 받으라는 거예요.

이건 실생활에서도 마찬가지잖아요.
메모장에 뭔가 쓰는 건 자유롭게 하지만,
중요한 계약서에 서명하기 전에는 한번 더 확인하잖아요.

에이전트에게도 같은 기준을 적용하는 겁니다.

#### 슬라이드 4: 연구와 정보 수집

[직장인 핵심 — 구조화된 검색 (3분)]

📌 화면: 연구와 정보 수집 프롬프트 코드 블록

자, 직장인분들이 가장 활용하기 좋은 부분이에요.
**구조화된 검색 프롬프트**입니다.

복잡한 조사 업무를 할 때 이런 프롬프트를 써보세요.

"이 정보를 구조화된 방식으로 검색하세요.
데이터를 수집하면서 여러 경쟁 가설을 개발하세요.
신뢰 수준을 추적하세요.
접근 방식과 계획을 정기적으로 자기 비판하세요."

이게 왜 효과적이냐면요.
그냥 "조사해줘"라고 하면 Claude가 대충 훑어보고 끝내거든요.
하지만 이렇게 구조를 잡아주면
**가설을 세우고, 검증하고, 수정하는** 과정을 거쳐요.
훨씬 깊이 있는 결과가 나옵니다.

실무 예를 들어볼게요.
"우리 서비스의 이탈률이 높은 원인을 조사해줘"
이렇게만 하면 일반적인 답이 나와요.

대신 이렇게 해보세요.
"이탈률 원인에 대해 3가지 이상의 가설을 세우고,
각 가설의 근거와 반증을 조사해서,
신뢰 수준(높음/중간/낮음)과 함께 정리해줘."

결과의 품질이 확연히 달라집니다.

#### 슬라이드 5: 서브에이전트 조율

[서브에이전트 — 위임의 기술 (2분)]

📌 화면: 서브에이전트 조율 섹션 + 경고 Callout

Claude의 최신 모델은 작업을 위임하는 데도 뛰어나요.
복잡한 작업을 **서브에이전트에게 나눠서** 동시에 처리할 수 있어요.

그런데 주의할 점이 있어요.
Claude Opus 4.6은 서브에이전트에 대한 **강한 선호**가 있어서,
간단한 작업인데도 서브에이전트를 만들려고 할 수 있어요.

그래서 이런 가이드를 주는 게 좋습니다.
"병렬로 실행될 수 있는 독립적인 작업에서만 서브에이전트를 사용하세요.
간단한 작업이나 순차적 작업은 직접 하세요."

모든 일을 위임하는 게 능사는 아니에요.
간단한 건 직접 하는 게 빠르잖아요.
사람도 마찬가지고, AI도 마찬가지예요.

#### 슬라이드 6: 에이전트 코딩 팁

[실용 팁 — 에이전트 제어 (2분)]

📌 화면: 에이전트 코딩 팁 섹션

에이전트를 활용할 때 실용적인 팁 세 가지예요.

첫 번째, **파일 생성을 줄이세요.**
에이전트가 작업하다 보면 임시 파일을 많이 만들어요.
"작업 종료 시 임시 파일을 정리하세요"라고 지시하세요.

두 번째, **과열함을 방지하세요.**
"요청된 것 이상의 기능 추가, 리팩토링, 개선을 하지 마세요."
이 지시가 없으면 Claude가 요청하지도 않은 것까지 고치려고 합니다.

세 번째, **환각을 최소화하세요.**
"열어보지 않은 코드에 대해 추측하지 마세요.
답변하기 전에 반드시 해당 파일을 읽으세요."
이러면 Claude가 추측 대신 실제 확인을 먼저 합니다.

#### 슬라이드 7: 마무리와 다음 강의 브릿지

[마무리 — 정리와 전환 (1분)]

오늘 배운 걸 정리하면요.

하나, 에이전트는 **사고와 행동의 반복 루프**입니다.
자율적으로 일하지만, 안전 가드레일이 필수예요.

둘, **가역성**을 기준으로 자율성을 조절하세요.
되돌릴 수 있는 건 자유롭게, 되돌리기 어려운 건 확인받고.

셋, 조사 업무에서는 **구조화된 검색 프롬프트**가 핵심이에요.
가설을 세우고, 검증하고, 신뢰 수준을 추적하게 하세요.

오늘의 액션 하나.
다음에 복잡한 조사를 요청할 때
"가설을 세우고 신뢰 수준과 함께 정리해줘"를 붙여보세요.

다음 시간에는 **기능별 팁**을 다룹니다.
비전 기능, 프론트엔드 디자인, 수정자 프레이밍 등
각 기능을 세밀하게 조정하는 실용적인 팁이에요.
기대해 주세요.

수고하셨습니다.
