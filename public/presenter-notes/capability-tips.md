#### 슬라이드 2: 향상된 비전 기능

[비전 — 이미지 처리 팁 (2분)]

📌 화면: 향상된 비전 기능 섹션

Claude Opus 4.5와 4.6은 이미지 처리 능력이 많이 좋아졌어요.
특히 여러 이미지를 동시에 다룰 때 성능이 향상되었습니다.

여기서 재미있는 팁 하나.
Claude에게 **자르기(crop) 도구**를 제공하면 성능이 더 높아져요.

왜 그럴까요?
사람도 서류를 볼 때 전체를 한눈에 보는 것보다
특정 부분을 확대해서 보는 게 더 정확하잖아요.
AI도 마찬가지예요.
이미지의 관련 영역을 "확대"할 수 있을 때
일관되게 더 나은 성능을 보여줬다고 합니다.

실무에서는
영수증 데이터 추출, 계약서 분석, 디자인 검토 같은 작업에서
이미지를 그냥 통째로 주는 것보다
관련 부분을 잘라서 주는 게 더 정확한 결과를 얻을 수 있어요.

#### 슬라이드 3: 프론트엔드 디자인 — "AI 슬롭" 방지

[프론트엔드 — AI 슬롭 탈출 (3분)]

📌 화면: 프론트엔드 디자인 섹션 + frontend_aesthetics 코드 블록

여러분 혹시 AI로 웹사이트를 만들어 보신 적 있나요?

만들어 보면 느끼는 게 있어요.
"어? 다 비슷하게 생겼네?"

흰 배경에 보라색 그라데이션,
Inter 폰트,
둥근 카드 레이아웃...

이걸 **"AI 슬롭"**이라고 부릅니다.
AI가 가장 일반적인, 가장 평균적인 디자인으로 수렴하는 현상이에요.

왜 이런 일이 생기냐면요.
AI는 학습 데이터의 **평균**으로 수렴하는 경향이 있거든요.
웹에서 가장 많이 보이는 디자인 패턴으로 기본 설정되는 거예요.

그래서 앤트로픽 공식 가이드에서 이런 프롬프트를 제안합니다.
"당신은 일반적이고 분포상의 출력으로 수렴하는 경향이 있습니다.
이를 피하세요. 놀랍고 만족스러운 창의적이고 독특한 프론트엔드를 만드세요."

구체적으로는
- 타이포그래피: Inter, Arial 같은 일반 폰트를 피하고 독특한 폰트 선택
- 색상: 소심하게 분배하지 말고, 날카로운 악센트와 주요 색상에 전념
- 모션: 고영향 순간에 애니메이션 집중
- 배경: 단색 기본값 대신 분위기와 깊이 만들기

이렇게 구체적으로 지시하면
AI 슬롭에서 벗어난 결과물을 얻을 수 있어요.

#### 슬라이드 4: 수정자 프레이밍

[핵심 — 수정자 프레이밍 (3분)]

📌 화면: 마이그레이션 고려사항 표 — 수정자 프레이밍 행

자, 오늘의 핵심 기법입니다.
직장인분들이 바로 써먹을 수 있는 **수정자 프레이밍**이에요.

예시를 볼게요.

"대시보드 만들어줘."
이렇게 하면 어떤 결과가 나올까요?
기본적인 대시보드가 나와요. 무난하지만 특별하지 않은.

이제 **수정자**를 붙여보세요.
"대시보드를 만들어줘. **가능한 한 많은 기능과 인터랙션을 포함해줘.**"

이 한 문장이 추가됐을 뿐인데
결과물의 풍성함이 확 달라집니다.

이 패턴은 어디에든 적용할 수 있어요.

"보고서 써줘" → "보고서를 써줘. 데이터 시각화와 구체적 사례를 풍부하게 포함해줘."
"기획서 작성해줘" → "기획서를 작성해줘. 실행 가능한 액션 플랜과 타임라인을 상세하게 포함해줘."
"이메일 써줘" → "이메일을 써줘. 상대방이 바로 의사결정할 수 있도록 옵션과 추천을 포함해줘."

핵심은 **기본 요청 + "어떻게 더 풍성하게" 수정자**의 조합이에요.

#### 슬라이드 5: 마이그레이션 고려사항 종합

[마이그레이션 종합 정리 (2분)]

📌 화면: 마이그레이션 고려사항 표 전체

Claude 4.6 모델로 마이그레이션할 때
알아두면 좋은 것들을 정리해 볼게요.

하나, **원하는 동작을 구체적으로** 설명하세요.
"잘 써줘"가 아니라 "출력에서 정확히 무엇을 보고 싶은지" 말해주세요.

둘, **수정자 프레이밍**을 활용하세요. 방금 배운 거죠.

셋, 애니메이션이나 인터랙티브 요소는 **명시적으로** 요청하세요.
말하지 않으면 넣지 않아요.

넷, 사고 구성은 **적응형 사고 + 노력 매개변수**로.

다섯, **사전 채움은 더 이상 지원되지 않아요.**
지난 시간에 배웠죠.

여섯, Claude 4.6은 **훨씬 적극적**이라
예전의 공격적인 표현을 **완화**해야 합니다.

#### 슬라이드 6: 마무리와 다음 강의 브릿지

[마무리 — 정리와 전환 (1분)]

오늘 배운 걸 정리하면요.

하나, 이미지 처리는 **관련 부분을 확대**해서 주면 더 정확해요.

둘, AI 슬롭을 방지하려면 **구체적인 디자인 지시**가 필요합니다.
폰트, 색상, 모션, 배경까지 지정해 주세요.

셋, **수정자 프레이밍**이 핵심이에요.
기본 요청에 "어떻게 더 풍성하게"를 붙이는 것.

오늘의 액션 하나.
다음에 Claude에게 뭔가를 요청할 때
기본 요청 뒤에 수정자 한 줄을 붙여보세요.
"~를 포함해줘", "~를 풍부하게 담아줘" 이런 식으로요.

다음 시간은 **마이그레이션 가이드**, 이 강의의 마지막 시간이에요.
모델이 바뀌면 프롬프트도 달라져야 한다는 이야기를 하겠습니다.
기대해 주세요.

수고하셨습니다.
