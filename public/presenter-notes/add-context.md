# 발표자 노트: 맥락을 추가하여 성능 향상

#### 슬라이드 1: 오프닝 — 공감과 문제 제기

지난 시간에 "명확하고 직접적으로"를 배웠죠.
프롬프트 공식도 배웠고요.
작업 + 대상 + 포함 요소 + 형식 + 톤.
이 다섯 칸을 채우면 결과가 확 좋아진다고 했어요.

오늘은 거기에 여섯 번째 칸을 추가합니다.
오늘의 주제는요, 한 글자입니다. "왜."

여러분 이런 경험 있으신가요?
회사에서 선배가 갑자기 다가와서
"이거 정리해줘"라고만 말하고 돌아가요.

그 순간 여러분 머릿속에 뭐가 떠오르나요?
질문이 열 개는 생기죠.

그런데 만약 선배가 이렇게 말했다면?
"내일 팀장님 보고에 쓸 건데, 예산 초과 원인 위주로 정리해줘."
감이 딱 오잖아요.

달라진 게 뭘까요? 바로 "왜"가 추가된 거예요.
AI도 완전히 똑같습니다.

#### 슬라이드 2: 단어 해부 — "맥락"과 "동기"

📌 화면: 페이지 제목 "맥락을 추가하여 성능 향상"

이 제목을 뜯어볼게요.

맥락. 영어로는 Context.
Con은 "함께", texere는 라틴어로 "짜다, 엮다".
함께 엮인 것, 정보들이 서로 짜여진 것.

한국어 "맥락"도 재밌어요.
맥(脈)은 핏줄, 락(絡)은 연결.
정보와 정보 사이를 연결하는 핏줄 같은 것.

피가 안 통하면 몸이 못 움직이잖아요?
맥락이 없으면 정보가 죽어 있는 거예요.

📌 화면: 공식문서 첫 문장

앤트로픽이 '동기(motivation)'라는 단어를 일부러 추가한 건
배경 정보가 아니라 여러분의 의도와 목적까지 알려달라는 뜻.

오늘의 핵심 공식:
AI한테 "왜"를 설명하면 AI가 "어떻게"를 판단합니다.

#### 슬라이드 3: 한 줄의 맥락 = 열 줄의 지시

"타겟팅된 응답"이라는 표현에 주목.
맥락 없으면 산탄총, 있으면 저격총.

놀라운 건, 이 전환에 단 한 줄이면 충분하다는 것.

📌 화면: Callout 블록

"한 줄의 맥락이 열 줄의 지시를 대체할 수 있습니다."

프롬프트 공식 다섯 칸을 다 채우면 꽤 길어지는데
맥락 한 줄이면 대체 가능.

"마케팅팀 상반기 캠페인 데이터를 분석해줘.
다음 달 예산 회의에서 '이 채널에 예산을 더 투입해야 한다'는 근거를 만들어야 해."

형식도, 톤도 지정 안 했는데
맥락이라는 핏줄 한 줄이 나머지를 전부 유추하게 해줌.

#### 슬라이드 4: 맥락의 4가지 종류

"맥락을 추가하라는 건 알겠는데 어떤 맥락을?"

하나의 "보고서 써줘"에 맥락을 하나씩 쌓으며 보여주기:

1. 용도 — "경영 회의에서 5분 발표할 보고서"
2. 대상 — "CTO 주관" → 기술 투자 강조
3. 워크플로우 위치 — "첫 번째 초안" vs "최종 검토"
4. 성공 기준 — "3분 안에 의사결정 가능하면 성공"

하나만 추가해도 달라지고, 두세 개 조합하면 극적으로 달라진다.

#### 슬라이드 5: TTS 예시 — Before/After

📌 화면: TTS 탭 비교

빨간 탭: "줄임표를 절대 사용하지 마"
→ 규칙은 따르지만 일반화 못함

초록 탭: "TTS 엔진이 발음을 못 하니까"
→ 동기를 알면 AI가 일반화 가능
"줄임표뿐 아니라 다른 특수문자도 조심해야겠다"

이게 맥락의 진짜 힘.
규칙이 아니라 원리를 알려줘서 AI가 스스로 판단하게 만드는 것.

#### 슬라이드 6: 이메일 요약 — Before/After

📌 화면: 이메일 탭 비교

빨간 탭: "이 이메일 스레드 요약해줘"

지난 시간 황금률 테스트:
"옆 팀 동료한테 '이거 요약해줘'라고 하면 뭐라고 물을까요?"
→ "누구한테 보여줄 건데?" 바로 그게 맥락.

초록 탭에는 맥락 3개가 숨어 있어요:
- 용도: 임원 브리핑용
- 대상: CFO
- 성공 기준: 5분 안에 파악

#### 슬라이드 7: 실무 적용 — 이메일, 보고서, 기획서

📌 화면: 실무 사례 카드들

이메일 — "대상"의 힘
"3년째 함께 일하는 파트너사 PM에게, 신뢰 관계 유지하고 싶어"
→ 톤이 완전히 바뀜

보고서 — "용도 + 워크플로우 위치 + 성공 기준"의 힘
"CTO 주관 경영 회의 5분 발표 최종 자료, 기술 투자→매출 성장 근거"

기획서 — "대상 + 성공 기준"의 힘
"인사팀장, 이직률 15% 증가, '투자하면 이직률 낮아질까?'"

📌 화면: 데이터 분석 탭
데이터 분석도 같은 원리.
"예산 증액 근거를 만들어야 해" 한 줄이 방향을 바꿈.

#### 슬라이드 8: 마무리 — 정리와 통찰

배운 것 정리:
1. 맥락 = 핏줄, 연결되어야 정보가 살아남
2. 4종류: 용도, 대상, 워크플로우 위치, 성공 기준
3. 프롬프트 공식 다섯 칸 + "왜"라는 여섯 번째 칸
   → 이 한 칸이 나머지 다섯을 대신할 수 있다

사실 사람한테도 똑같이 적용되는 원칙.
"이거 해줘"는 명령, "이런 이유 때문이야"는 협업.
