# 발표자 노트: 도구 사용

#### 슬라이드 1: 도입부 — "제안 vs 실행"

[오프닝 — 브릿지와 핵심 질문 (2분)]

지난 시간에 출력의 형식을 제어하는 방법을 배웠죠.
"하지 말라" 대신 "무엇을 하라"로 지시하는 것.
프롬프트 스타일이 출력에 영향을 준다는 것.

오늘은 한 단계 더 나아갑니다.
AI가 텍스트를 생성하는 것을 넘어서
**실제로 행동하게 만드는 법**이에요.

📌 화면: "도구 사용" 페이지 제목

먼저 여러분에게 질문 하나 드릴게요.

여러분이 동료에게 "이 코드 좀 개선할 수 있을까?"라고 물으면
동료가 뭐라고 하겠어요?
"응, 이렇게 바꾸면 좋을 것 같아"라고 **제안만** 하겠죠.

그런데 "이 코드 직접 수정해줘"라고 하면?
실제로 코드를 **수정해서** 줄 거예요.

AI도 정확히 같습니다.
물어보면 제안하고, 시키면 실행해요.

📌 화면: 직장인 학습 가이드 Callout

API를 직접 다루시는 개발자분이 아니시라면
오늘 강의에서 핵심은 딱 하나예요.
**"제안 vs 실행"의 차이**를 이해하는 것.

이 원칙은 Claude.ai든, ChatGPT든, Copilot이든
모든 AI 도구에 적용됩니다.

#### 슬라이드 2: 명시적 지시의 차이 — 탭 비교

[핵심 — 제안 vs 실행 (3분)]

📌 화면: 탭 비교 — 제안만 함 / 변경을 수행함 / 훨씬 더 직접적

세 가지 프롬프트를 비교해 볼게요.

첫 번째. "이 함수를 개선할 변경 사항을 제안해줄 수 있나요?"
이렇게 물어보면 Claude는 정중하게 제안만 합니다.
"이런 식으로 바꾸면 좋을 것 같습니다" 하고요.

두 번째. "이 함수의 성능을 개선하도록 수정해줘."
이렇게 하면 실제로 코드를 수정합니다.

세 번째. "인증 흐름에 이 편집 사항을 적용해줘."
이건 더 직접적이에요. 어디에, 무엇을, 하라는 건지 명확하죠.

차이가 보이시나요?
"~할 수 있나요?"는 가능성을 묻는 거고,
"~해줘"는 행동을 지시하는 거예요.

실무에서 이런 차이가 많이 나타나거든요.
코드 리뷰할 때 "수정 사항을 제안해줘"라고 하면 코멘트만 달고,
"직접 수정해줘"라고 하면 실제로 코드를 고쳐줍니다.

#### 슬라이드 3: Claude를 더 적극적으로 / 더 신중하게

[적극성 조절 (3분)]

📌 화면: 적극적으로 만들기 프롬프트 코드 블록

그런데 매번 "직접 해줘"라고 쓰기 귀찮을 수 있잖아요.
시스템 프롬프트에서 기본 모드를 설정할 수 있어요.

적극적으로 만들고 싶으면?
"기본적으로 제안만 하지 말고 변경 사항을 직접 구현하세요."
이 한 줄이면 Claude가 항상 행동 우선으로 움직입니다.

📌 화면: 신중하게 만들기 프롬프트 코드 블록

반대로 신중하게 만들고 싶으면?
"명확하게 변경하라는 지시가 없는 한 구현에 뛰어들지 마세요."
이러면 Claude가 항상 확인을 먼저 합니다.

여기서 중요한 주의사항이 있어요.

📌 화면: 과다 트리거 경고 Callout

Claude Opus 4.5와 4.6은 이전 모델보다 시스템 프롬프트에 **더 민감하게** 반응합니다.
예전에 도구를 더 잘 쓰라고 넣었던 강한 표현들.
"중요: 반드시 이 도구를 사용해야 합니다..." 이런 거요.
이런 표현이 이제는 **과다 트리거**를 유발할 수 있어요.
너무 공격적으로 도구를 남발하는 거죠.

그래서 이전 모델을 위해 작성했던 프롬프트를 쓰고 계시다면
표현을 좀 완화하는 게 좋습니다.
"반드시 사용해야 합니다" → "사용하세요" 정도로요.

#### 슬라이드 4: 병렬 도구 호출

[병렬 실행 — 속도 최적화 (2분)]

📌 화면: 병렬 도구 호출 최적화 섹션

API를 다루시는 분들을 위한 내용인데요.
Claude의 최신 모델은 **여러 도구를 동시에** 실행할 수 있어요.

예를 들어 파일 3개를 읽어야 할 때,
하나 읽고, 다음 하나 읽고, 또 다음 하나 읽고... 이렇게 순차적으로 하면 느리잖아요.
3개를 동시에 읽으면 3배 빨라지는 거죠.

이걸 최대한 활용하려면
"독립적인 도구 호출을 병렬로 수행하세요"라고 지시하면 되고,
반대로 안정성이 중요하면
"순차적으로 실행하세요"라고 하면 됩니다.

#### 슬라이드 5: 마무리와 다음 강의 브릿지

[마무리 — 정리와 전환 (1분)]

오늘의 핵심을 정리하면요.

하나, **"~할 수 있나요?" → "~해줘"**로 바꾸세요.
물어보면 제안하고, 시키면 실행합니다.

둘, 시스템 프롬프트로 **기본 적극성 수준**을 설정할 수 있어요.
적극적 모드와 신중한 모드, 업무 성격에 맞게 선택하세요.

셋, 최신 모델은 프롬프트에 **더 민감**합니다.
예전의 강한 표현을 완화하는 것, 잊지 마세요.

오늘의 액션 하나.
다음에 Claude를 쓸 때
"이거 어떻게 하면 좋을까?"라고 물어보는 대신
"이거 직접 해줘"라고 써보세요.
결과가 확 달라질 거예요.

다음 시간에는 **사고와 추론**을 다룹니다.
AI가 행동하기 전에 "생각"하게 만드는 방법이에요.
행동만 빠르면 실수할 수 있잖아요.
생각하고 행동하게 만드는 법, 기대해 주세요.

수고하셨습니다.
