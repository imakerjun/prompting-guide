## 과도한 사고와 지나친 철저함

Claude Opus 4.6은 이전 모델보다 훨씬 더 많은 사전 탐색을 수행합니다. 이전에 모델이 더 철저하도록 장려하는 프롬프트를 사용했다면 조정이 필요합니다:

- **포괄적인 기본값을 더 타겟팅된 지시로 교체.** `"기본적으로 [도구]를 사용해"` 대신 → `"문제 이해를 높일 때 [도구]를 사용해"`
- **과도한 프롬프팅 제거.** `"의심스러우면 [도구]를 사용해"` 같은 지시는 과다 트리거를 유발합니다.
- **노력(effort)을 대안으로 사용.** Claude가 계속 과도하게 적극적이라면, 더 낮은 노력 설정을 사용하세요.

```
문제에 접근하는 방법을 결정할 때, 하나의 접근 방식을 선택하고
그것에 전념하세요. 추론에 직접 모순되는 새로운 정보를 만나지
않는 한 결정을 재검토하지 마세요. 두 가지 접근 방식을
저울질하고 있다면, 하나를 선택하고 끝까지 해보세요.
선택한 접근 방식이 실패하면 나중에 궤도를 수정할 수 있습니다.
```

## 사고 및 인터리브 사고 기능 활용

Claude의 최신 모델은 도구 사용 후 반성이나 복잡한 다단계 추론에 특히 도움이 되는 사고 기능을 제공합니다.

<Callout emoji="🧠">
Claude Opus 4.6은 **적응형 사고** (`thinking: {type: "adaptive"}`)를 사용합니다. 내부 평가에서 적응형 사고는 확장 사고보다 안정적으로 더 나은 성능을 보여줍니다.
</Callout>

Claude는 두 가지 요소를 기반으로 사고를 보정합니다:
1. **노력 매개변수**: 높은 노력 → 더 많은 사고
2. **쿼리 복잡성**: 더 복잡한 쿼리 → 더 많은 사고

사고가 필요하지 않은 쉬운 쿼리에서는 모델이 직접 응답합니다.

### 사고 동작 안내

```
도구 결과를 받은 후, 그 품질을 신중하게 반성하고 진행하기
전에 최적의 다음 단계를 결정하세요. 사고를 사용하여 이
새로운 정보를 기반으로 계획하고 반복한 다음, 최선의 다음
행동을 취하세요.
```

### 사고 빈도 줄이기

```
확장 사고는 지연 시간을 추가하며, 답변 품질을 의미 있게
향상시킬 때만 사용해야 합니다 — 일반적으로 다단계 추론이
필요한 문제에서. 의심스러우면, 직접 응답하세요.
```

## 사고를 위한 팁

| 팁 | 설명 |
|----|------|
| 일반적 지시 우선 | `"철저하게 생각해"`가 손으로 작성한 단계별 계획보다 더 나은 추론을 생산합니다 |
| 멀티샷 + 사고 | few-shot 예시에 `<thinking>` 태그를 사용하여 추론 패턴을 보여주세요 |
| 수동 CoT 대안 | 사고가 꺼져 있을 때 `<thinking>`과 `<answer>` 태그로 추론과 출력을 분리 |
| 자기 점검 | `"끝내기 전에, [테스트 기준]에 대해 답변을 검증하세요."` 추가 |

<Callout emoji="⚠️">
확장 사고가 비활성화되어 있을 때, Claude Opus 4.5는 **"think"라는 단어에 특히 민감**합니다. `"consider"`, `"evaluate"`, `"reason through"` 같은 대안을 사용하세요.
</Callout>
