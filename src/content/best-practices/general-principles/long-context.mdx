# 긴 컨텍스트 프롬프팅

<OriginalDocLink href="https://docs.anthropic.com/en/docs/build-with-claude/prompt-engineering/claude-prompting-best-practices#long-context-prompting">공식문서에서 보기</OriginalDocLink>

대용량 문서(보통 A4 기준 15-20페이지 이상)나 데이터가 풍부한 입력으로 작업할 때는 프롬프트를 신중하게 구조화해야 최상의 결과를 얻을 수 있습니다.

Claude는 한 번에 수백 페이지 분량의 텍스트를 처리할 수 있지만, **문서를 어떻게 배치하고 질문하느냐**에 따라 응답 품질이 크게 달라집니다. 마치 두꺼운 보고서를 읽을 때 목차부터 확인하고, 필요한 부분을 먼저 찾는 것처럼 — Claude에게도 정보를 체계적으로 제공해야 합니다.

아래 세 가지 핵심 원칙을 따르면 긴 문서 작업의 정확도와 효율을 크게 높일 수 있습니다.

## 1. 긴 데이터를 상단에, 질문을 하단에 배치

긴 문서와 입력을 프롬프트 상단에 배치하고, 쿼리나 지시 사항은 프롬프트 하단에 넣으세요.

이유는 간단합니다. Claude는 프롬프트를 순서대로 처리하기 때문에, 먼저 전체 문서를 읽은 후 질문을 만나면 **이미 파악한 내용을 바탕으로 정확하게 답변**할 수 있습니다. 반대로 질문이 먼저 오면, 문서를 읽으면서 무엇이 중요한지 판단해야 하므로 핵심을 놓칠 가능성이 높아집니다.

<Callout emoji="📊">
쿼리를 끝에 배치하면 테스트에서 응답 품질이 **최대 30%까지** 향상될 수 있습니다. 특히 복잡한 다중 문서 입력에서 효과적입니다.
</Callout>

**실무 적용 예시:**

<Tabs items={['❌ 질문이 위에', '✅ 질문이 아래에']}>
  <Tabs.Tab>
```
Q3 매출 하락 원인을 분석해줘.

<document>
{50페이지 분량의 분기 보고서}
</document>
```

Claude가 "매출 하락 원인"을 찾으면서 50페이지를 읽어야 해서, 관련 없는 정보에 주의가 분산됩니다.
  </Tabs.Tab>
  <Tabs.Tab>
```
<document>
{50페이지 분량의 분기 보고서}
</document>

위 보고서에서 Q3 매출 하락의 원인을 분석해줘.
주요 원인 3가지를 근거와 함께 정리해줘.
```

Claude가 먼저 보고서 전체를 파악한 뒤, 매출 하락과 관련된 정보를 정확히 추출합니다.
  </Tabs.Tab>
</Tabs>

## 2. XML 태그로 문서 콘텐츠와 메타데이터를 구조화

여러 문서를 사용할 때, 각 문서를 `<document>` 태그로 감싸고 `<document_content>`와 `<source>` 하위 태그를 사용하세요.

태그 없이 여러 문서를 나열하면 Claude가 **어디서부터 어디까지가 하나의 문서인지**, **어떤 문서에서 나온 정보인지** 혼동할 수 있습니다. XML 태그로 각 문서의 경계와 출처를 명확히 하면, Claude가 정보를 정확히 추적하고 인용할 수 있습니다.

```xml
<documents>
  <document index="1">
    <source>2025년_1분기_매출보고서.pdf</source>
    <document_content>
    {문서 1 내용}
    </document_content>
  </document>
  <document index="2">
    <source>2025년_2분기_매출보고서.pdf</source>
    <document_content>
    {문서 2 내용}
    </document_content>
  </document>
</documents>

위 두 보고서를 비교하여 분기별 매출 변화를 분석해줘.
각 수치의 출처 문서를 명시해줘.
```

이 구조는 특히 **계약서 여러 개 비교**, **월간 회의록 분석**, **규정 간 충돌 확인** 등 다중 문서 작업에서 필수적입니다.

## 3. 인용문으로 응답을 근거에 기반하게 만들기

긴 문서 작업의 경우 Claude에게 작업 수행 전에 문서의 관련 부분을 **먼저 인용하도록** 요청하세요.

이 테크닉이 효과적인 이유는 두 가지입니다. 첫째, Claude가 관련 부분을 직접 찾아 인용하는 과정에서 **나머지 문서의 노이즈를 자연스럽게 걸러냅니다**. 둘째, 인용이 포함된 답변은 **검증이 쉽습니다** — 원문과 대조하여 정확성을 바로 확인할 수 있죠.

```
아래 규정집에서 재택근무 관련 조항을 모두 찾아서,
먼저 해당 조항 원문을 인용한 뒤 요약해줘.

<document>
{200페이지 분량의 사내 규정집}
</document>
```

이렇게 하면 Claude는 "제3장 제15조에 따르면..." 식으로 근거를 제시하면서 답변하므로, **할루시네이션(사실과 다른 내용 생성) 위험을 크게 줄일** 수 있습니다.

## 실무에서 이렇게 쓰세요

긴 컨텍스트 프롬프팅은 **대량의 문서를 다루는 실무 업무**에서 특히 강력합니다.

<UseCaseCard
  emoji="📑"
  title="100페이지 계약서 검토"
  description="긴 문서를 상단에 배치하고, 질문을 하단에 넣어 핵심 조항을 정확히 찾습니다"
  prompt={`<contract>
{100페이지 분량의 계약서 전문}
</contract>

위 계약서에서 다음 사항을 검토해줘:
1. 계약 해지 조건 (위약금 포함)
2. 지적재산권 귀속 조항
3. 우리에게 불리할 수 있는 조항

각 항목에 대해 해당 조항 원문을 인용한 뒤 요약해줘.`}
/>

<UseCaseCard
  emoji="📊"
  title="1년치 회의록 분석"
  description="XML 태그로 각 회의를 구분하면, Claude가 시간 순서와 맥락을 정확히 파악합니다"
  prompt={`<meetings>
  <meeting date="2025-01">
    <attendees>김팀장, 이과장, 박대리</attendees>
    <content>{1월 회의록}</content>
  </meeting>
  <meeting date="2025-02">
    <attendees>김팀장, 이과장</attendees>
    <content>{2월 회의록}</content>
  </meeting>
  ...12개월분 반복...
</meetings>

위 1년치 회의록을 분석해서:
1. 반복적으로 논의되었지만 해결되지 않은 이슈 Top 3
2. 주요 의사결정 타임라인
3. 다음 분기에 우선 해결해야 할 사항 추천`}
/>

<UseCaseCard
  emoji="📋"
  title="규정집에서 특정 조항 찾기"
  description="인용을 먼저 요청하면 할루시네이션 없이 정확한 근거를 얻을 수 있습니다"
  prompt={`<regulations>
{사내 인사규정 전문 - 150페이지}
</regulations>

위 규정에서 "육아휴직" 관련 모든 조항을 찾아줘.

작업 방식:
1. 먼저 관련 조항의 원문을 정확히 인용해줘
2. 각 조항을 쉬운 말로 요약해줘
3. 마지막에 육아휴직 신청 시 필요한 절차를 단계별로 정리해줘`}
/>

## 연습문제

### 문제: 다중 문서 비교 분석 프롬프트 구성

<Quiz
  hint="문서를 XML 태그로 구조화하고, 질문은 문서 아래에 배치하세요. 인용도 요청하면 더 좋습니다."
  answer={<>
    <pre><code>{`<documents>\n  <document index="1">\n    <source>A사 제안서</source>\n    <document_content>{A사 제안서 내용}</document_content>\n  </document>\n  <document index="2">\n    <source>B사 제안서</source>\n    <document_content>{B사 제안서 내용}</document_content>\n  </document>\n</documents>\n\n위 두 제안서를 비교 분석해줘.\n비교 기준: 가격, 기술력, 납기, 사후지원\n각 기준에 대해 해당 제안서의 관련 부분을 인용한 뒤 평가해줘.\n마지막에 종합 추천 의견을 달아줘.`}</code></pre>
    <p>세 가지 원칙을 모두 적용했습니다: (1) 문서를 상단에 배치, (2) XML 태그로 구조화, (3) 인용 요청으로 근거 기반 답변 유도.</p>
  </>}
>
A사와 B사의 제안서를 비교 분석하는 프롬프트를 작성하세요. 이 페이지에서 배운 **세 가지 원칙**(문서 상단 배치, XML 태그 구조화, 인용 요청)을 모두 적용해보세요.
</Quiz>
