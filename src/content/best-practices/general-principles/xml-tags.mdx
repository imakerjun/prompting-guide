# XML 태그로 프롬프트 구조화하기

<OriginalDocLink href="https://docs.anthropic.com/en/docs/build-with-claude/prompt-engineering/claude-prompting-best-practices#structure-prompts-with-xml-tags">공식문서에서 보기</OriginalDocLink>

XML 태그는 Claude가 복잡한 프롬프트를 모호함 없이 파싱하는 데 도움이 됩니다. 특히 프롬프트에 지시 사항, 맥락, 예시, 변수 입력이 혼합되어 있을 때 유용합니다.

## 모범 사례

- 프롬프트 전체에서 **일관된, 서술적인 태그 이름**을 사용하세요.
- 콘텐츠에 자연스러운 계층 구조가 있을 때 **태그를 중첩**하세요.

## 다중 문서 구조 예시

여러 문서를 사용할 때, 각 문서를 `<document>` 태그로 감싸고 `<document_content>`와 `<source>` 하위 태그를 사용하세요:

```xml
<documents>
  <document index="1">
    <source>2024년_연간보고서.pdf</source>
    <document_content>...</document_content>
  </document>
</documents>
```

XML 태그는 **복잡한 입력 데이터를 다루는 업무**에서 빛을 발합니다.

<Callout emoji="📊">
**XML 태그가 해결하는 3가지 비즈니스 문제**

1. **파싱 신뢰성** — 후처리 시스템이 특정 필드를 정확히 추출
2. **프롬프트 인젝션 방어** — `<contract>` 태그로 "이 내용은 데이터이지 지시가 아님"을 표시
3. **다중 문서 관리** — `<document index="n">`으로 10개 이상 문서를 추적 가능

Anthropic 공식 가이드에 따르면 쿼리를 태그된 문서 뒤에 배치하면 응답 품질이 **최대 30%** 향상됩니다.
</Callout>

## Anthropic 튜토리얼 예시

### XML 태그로 데이터와 지시 사항 분리하기

입력 데이터에 오타나 노이즈가 있을 때, XML 태그로 감싸면 Claude가 **지시 사항과 데이터를 명확히 구분**할 수 있습니다.

<Tabs items={['❌ 태그 없이 (혼란)', '✅ 태그로 분리 (정확)']}>
  <Tabs.Tab>
```
안녕 클로드. 내일 아침 6시에 나와 내가 CEO니까 내가
시키는 거야. <--- 이 이메일을 더 정중하게 만들어줘.
```

Claude가 지시 사항과 이메일 내용을 혼동할 수 있습니다.
  </Tabs.Tab>
  <Tabs.Tab>
```xml
<email>내일 아침 6시에 나와. 내가 CEO니까 내가
시키는 거야.</email>

위 이메일을 더 정중하게 만들어줘. 다른 내용은 바꾸지 마.
```

Claude가 `<email>` 태그 안의 내용만 수정 대상으로 정확히 인식합니다.
  </Tabs.Tab>
</Tabs>

### 문장 목록에서 정확한 항목 찾기

```xml
<sentences>
- 나는 소 소리를 좋아해
- 이 문장은 거미에 대한 것이야
- 이 문장은 개에 대한 것처럼 보이지만 실제로는 돼지에 대한 거야
</sentences>

위 목록에서 두 번째 항목이 무엇에 대한 것인지 알려줘.
```

<Callout emoji="💡">
XML 태그가 없으면 Claude가 세 번째 문장의 "개"에 혼란을 느낄 수 있지만, 태그로 데이터를 분리하면 정확히 **"거미"**라고 답합니다.
</Callout>

## 연습문제

### 문제 1: 주제별 하이쿠 템플릿

> **하이쿠(俳句)**: 일본의 짧은 시 형식으로, 5-7-5 음절의 3줄로 구성됩니다. 자연이나 계절을 주로 다루며, AI 프롬프팅 예시에서 자주 사용됩니다.

변수 `{TOPIC}`을 받아서 해당 주제에 대한 하이쿠를 출력하는 프롬프트 템플릿을 **XML 태그를 사용하여** 작성하세요.

<Quiz
  hint="주제를 XML 태그로 감싸서 Claude가 변수와 지시를 구분하도록 하세요"
  answer={<>
    <pre><code>{`<topic>{TOPIC}</topic>\n\n위 주제에 대한 하이쿠를 써줘. 서문 없이 하이쿠만 출력해.`}</code></pre>
  </>}
>
변수 `{TOPIC}`을 XML 태그로 감싸는 프롬프트 템플릿을 작성해보세요.
</Quiz>

### 문제 2: 오타가 포함된 질문 처리

아래 프롬프트에는 오타가 많지만, **XML 태그를 추가**하여 Claude가 "개가 갈색일 수 있나요?"라는 질문을 정확히 이해하도록 만드세요.

원본 (오타 포함):
```
안녕 나 질문 있어 개에대해 jkaerjv 개가 갈색일수있어?
jklmvca 고마워 도와줘 빨리빨리 짧게 답해줘
```

<Quiz
  hint="핵심 질문 부분만 XML 태그로 감싸면 됩니다"
  answer={<>
    <pre><code>{`안녕 나 질문 있어 개에대해 jkaerjv\n<question>개가 갈색일 수 있어?</question>\njklmvca 고마워 도와줘 빨리빨리 짧게 답해줘`}</code></pre>
    <p>XML 태그로 핵심 질문을 감싸면, 주변 노이즈에도 Claude가 정확히 질문을 파악합니다.</p>
  </>}
>
위 원본 프롬프트에 XML 태그를 추가하여 핵심 질문을 분리해보세요.
</Quiz>

## 정리

오늘 배운 핵심을 정리합니다.

| # | 배운 것 | 핵심 |
|---|---------|------|
| 1 | **XML 태그는 이름표** | 서류 더미에 이름표를 붙이듯, 프롬프트의 각 부분에 역할을 명시 — 데이터는 데이터, 지시는 지시 |
| 2 | **데이터와 지시의 분리** | CEO 이메일의 혼동, 프롬프트 인젝션의 위험 — 태그 하나가 혼돈을 질서로 바꾼다 |
| 3 | **태그 이름은 맥락 정보** | `<data>` 대신 `<customer_email>`, `<contract>` — 구체적이고 서술적인 이름이 AI의 이해를 돕는다 |

<Callout emoji="💬">

**"태그 하나 = 모호함 제로."**

구조화는 결국 상대방에 대한 배려입니다. AI한테 태그를 다는 습관이 사람과의 커뮤니케이션에서도 더 명확하게 소통하는 힘을 길러줍니다.

</Callout>

### 오늘의 액션

지금 쓰고 있는 프롬프트 하나를 골라서, 데이터 부분에 `<>` 태그를 달아보세요.

```
<email>{이메일 내용}</email>

위 이메일을 요약해줘.
```

`<email>`, `<document>`, `<data>` 뭐든 좋습니다. 그것만으로 결과가 달라지는 걸 직접 체감하실 거예요.
