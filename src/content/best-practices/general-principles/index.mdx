# 일반 원칙

<OriginalDocLink href="https://docs.anthropic.com/en/docs/build-with-claude/prompt-engineering/claude-prompting-best-practices#general-principles">공식문서에서 보기</OriginalDocLink>

프롬프트 엔지니어링의 기초가 되는 7가지 일반 원칙을 다룹니다. 이 원칙들은 Claude뿐 아니라 ChatGPT, Gemini 등 **어떤 AI 도구에서든 적용**할 수 있는 보편적인 커뮤니케이션 기법입니다.

각 원칙 페이지에는 **실무 사례**, **비교 예시**, **연습문제**가 포함되어 있어, 학습 후 바로 업무에 적용할 수 있습니다.

<Callout emoji="💡">
**먼저 읽으면 좋은 순서**: "명확하고 직접적으로" → "맥락을 추가하여 성능 향상" → "예시를 효과적으로 사용하기" 순서로 학습하면, 이 세 가지만으로도 프롬프트 품질이 극적으로 향상됩니다.
</Callout>

| 원칙 | 핵심 | 업무 활용 예시 |
|------|------|-------------|
| [명확하고 직접적으로](/docs/best-practices/general-principles/be-clear-and-direct) | 모호함을 없애고 정확하게 설명 | 이메일, 보고서, 데이터 분석 |
| [맥락을 추가하여 성능 향상](/docs/best-practices/general-principles/add-context) | 왜 필요한지 배경 설명 | 임원 브리핑, 고객 응대, 기획서 |
| [예시를 효과적으로 사용하기](/docs/best-practices/general-principles/use-examples) | 원하는 형식을 예시로 보여주기 | 브랜드 톤 유지, 리뷰 분류, 계약서 |
| [XML 태그로 프롬프트 구조화](/docs/best-practices/general-principles/xml-tags) | 지시, 맥락, 입력을 태그로 구분 | 제안서 비교, 주간 보고, 데이터 분리 |
| [Claude에게 역할 부여하기](/docs/best-practices/general-principles/give-a-role) | 전문가 역할을 지정 | 법률 검토, 마케팅 카피, 코드 리뷰 |
| [긴 컨텍스트 프롬프팅](/docs/best-practices/general-principles/long-context) | 대용량 문서 다루는 전략 | 계약서, 회의록, 규정집 분석 |
| [AI에게 정체성 설정하기](/docs/best-practices/general-principles/model-self-knowledge) | AI의 역할과 맥락을 고정 | 팀 전용 어시스턴트, 프로젝트 맥락 |
